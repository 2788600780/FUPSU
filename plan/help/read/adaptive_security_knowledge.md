# 自适应输入安全性 —— 完整知识

## 一、概念基础

### 1.1 反应式功能函数

UPSI/updPSU 是多轮交互协议，非一次性：

```
Round 1: init → 输出 I_1
Round 2: (add x1, del y1) → 输出 I_2
Round 3: (add x2, del y2) → 输出 I_3
...
```

每一轮的输入可以基于之前轮次观察到的输出来决定。

### 1.2 两个"adaptive"概念

| | 自适应腐败 | 自适应输入选择 |
|------|-----------|--------------|
| 问题 | 敌手在哪个 round 攻破谁 | 敌手根据之前 round 的观察动态选择下一轮输入 |
| UC 默认要求？ | 可选 | **是**——对反应式功能函数的默认要求 |

Liu et al. (Eurocrypt 2026)：**静态腐败 + 半诚实敌手**（只关注自适应输入选择）。

### 1.3 Selective vs Adaptive 安全定义

**Selective（之前所有 UPSI 论文用的）**：
- 所有 round 的输入在协议开始前一次性选定
- 输入不能根据上一轮输出动态调整

**Adaptive（UC 默认要求）**：
- 每轮输入可以基于之前轮次的 transcript 动态选择

## 二、不安全的根本条件

```
自适应输入不安全 ≡ ① 持久数据结构 + ② 公开位置映射 + ③ 固定容量约束

三个条件缺一不可：
  BMX22：     ① ✅ + ② ❌ → 安全
  BMS+24：    ① ✅ + ② ✅ + ③ ✅ → 被攻破
  LTS+26：    ① ❌ → 安全
  LMR+26(tree)：① ✅ + ② ✅ + ③ ✅ → 不安全
  LMR+26(okvs)：① ✅ + ② ❌ → 安全
  FUPSU：     ① ✅ + ② ❌ + ③ ❌ → 免疫
```

## 三、完整攻击流程

### 3.1 攻击目标：Path-ORAM 结构

BMS+24 和 ACG+24 使用 Path-ORAM 风格的二叉树编码集合：

```
                     [root]                           ← 第 0 层（1 个节点）
                    /      \
                 [A]        [B]                       ← 第 1 层（2 个节点）
                /    \      /    \
             [C]    [D]   [E]    [F]                  ← 第 2 层（4 个节点）
            /  \    /  \   /  \    /  \
          [G] [H] [I] [J] [K] [L] [M] [N]            ← 第 3 层 = 叶子层（8 个节点）

深度 d = log₂(N)，叶子数 N = 2^d
```

| 参数 | 含义 | 示例值 |
|------|------|--------|
| N | 叶子数 | 256（d = 8） |
| b | 每个节点最多存的元素数 | 5 |
| c | 一条 root-to-leaf 路径总容量 | b × d = 5 × 8 = 40 |

**元素存储规则**：

```
h(x) = 元素 x 映射到的叶子编号（公开函数）
x 存储在 root → h(x) 路径上的某个内部节点

例：h(x) = I（第 5 个叶子）
  → x 可以存在 root、B、E 或 I 这些节点中
```

**驱逐过程（公开可见）**：

```
新元素加入时：
  1. 放入 root 节点
  2. 随机选一条叶子路径作为"驱逐路径"（双方可见）
  3. 沿驱逐路径，元素尽量向下推到更深节点
  4. 推到叶子时，最老/多余的元素被逐出树
```

**正常 vs 恶意**：

```
正常情况：
  总元素数 n << N×b → 每条路径上元素远少于 c → 安全

恶意情况：
  敌手把 c+1 个元素全部映射到同一个叶子
  → 这些元素全部挤在同一条 root-to-leaf 路径上
  → c+1 > c → 该路径溢出 → 数据结构崩溃
```

h(·) 必须公开——双方都需要用它来探测对方的数据结构。这就是漏洞的根源。

### 3.2 攻击一：破坏正确性

```
1. 敌手观察 h(·)
2. 找到 c+1 个元素全部映射到同一叶子路径
   成本：O(N·c) ≈ 数千次哈希（秒级完成）
3. 让诚实方加入这些元素
4. 路径容量 c < c+1 → 溢出 → 协议崩溃
```

无需腐败任何一方。

### 3.3 攻击二：破坏隐私（区分 x vs x'）

**设置**：诚实方 P1 的集合只有一个元素——x 或 x'。

**第一层**：h(x) 和 h(x') 在根的不同侧 → 概率 1/2

**第二层**：找 c 个元素全部碰撞到 h(x) 路径 → 概率 1（总能找到）

**第三层**：驱逐路径命中 h(x) → 概率 c/N ≈ 12%

**第四层**：观察结果
- P1 有 x：c+1 元素 → 溢出 → abort
- P1 有 x'：c 元素刚好装满 → 成功

**总成本**：期望 17 次协议重试，多项式时间。

**区分优势**：≈ 1（一旦事件全发生，区分是确定的）。

### 3.4 自适应攻击如何运用于树结构 —— 逐轮分解

**核心**：敌手利用"观察 → 自适应选择输入 → 观察结果"的循环，将静态知识（碰撞集）转化为动态攻击（隐私推断）。

---

#### 预备阶段：构建武器库（协议开始前即可完成）

```
h(·) 是公开的。敌手在协议开始前可以：

对每个叶子 i ∈ [N]，暴力搜索找到 c+1 个元素，
全部映射到叶子 i

成本：N × (c+1) ≈ 256 × 31 ≈ 8000 次哈希 → 秒级完成
结果：武器库 = {对每个叶子，都有 c+1 个碰撞元素}

selective 模型：碰撞集提前选好了，但所有 round 的输入已固定 → 无法动态使用 → 攻击无效
adaptive 模型：碰撞集可以在后续 round 中按需取出 → 攻击开始
```

---

#### 攻击目标：区分 x vs x'

诚实方 P1 的集合只有一个元素——x 或 x'。敌手想知道是哪个。

---

#### Round 1：正常协议执行 + 观察

```
敌手观察：
  h(x) 和 h(x') 分别在哪个子树？

  如果 h(x) 和 h(x') 在根的同一侧 → 这个 epoch 放弃
  如果 h(x) 在左、h(x') 在右（或反过来）→ 继续

  概率 1/2，期望 2 个 epoch 后满足条件
```

**"自适应"体现在哪**：如果条件不满足，敌手**放弃这个 epoch**——选择一个无关的 delta，正常完成协议，等下一轮。在 selective 模型中做不到这一点——所有 round 的输入提前固定，不能"跳过"。

---

#### Round 2：自适应选择输入 ← 关键一步

```
基于 Round 1 的观察（h(x) 在左子树），敌手：

从武器库中取出"叶子 h(x) 的碰撞集 S"
  S = {s1, s2, ..., sc}  共 c 个元素，全部映射到 h(x) 路径

敌手让诚实方 P1 把 S 加入集合（正常协议操作，不违规）
```

**"自适应"体现在哪**：敌手选择的 S 不是提前固定的——它是基于 Round 1 观察到的 h(x) 位置来**动态选择**的。如果 h(x) 在叶子 17，就拿叶子 17 的碰撞集；如果在叶子 200，就拿叶子 200 的。

---

#### Round 3：观察驱逐结果 + 推断

```
Path-ORAM 中，新元素从 root 向下推（eviction）。
驱逐路径随机选择（公开可见）。

敌手观察驱逐路径：
  如果 h(x) 的叶子路径 ≠ 驱逐路径 → 无事发生，下个 epoch 重试
    概率：1 - c/N ≈ 88%

  如果 h(x) 的叶子路径 = 驱逐路径 → 攻击生效！
    概率：c/N ≈ 12%

    此时：
    · P1 有 x → 路径上有 S 的 c 个 + x = c+1 个 → 溢出 → abort
    · P1 有 x' → S 的 c 个刚好装满 → 正常完成
    
    推断：
      abort  → P1 持有 x
      正常  → P1 持有 x'
```

---

#### 总成本

```
每轮成功概率 ≈ 1/2（第一层）× c/N（第三层）= c/(2N)

当 N=256, c=30：c/(2N) = 30/512 ≈ 6%

期望轮数 ≈ 17 个 epoch
每次都执行正常的协议（不违规，半诚实）
```

---

#### 为什么 selective 模型下攻击失败

```
selective 要求：所有 round 的 delta 在 protocol 开始时一次性选定

但敌手在第 0 轮时不知道：
  - h(x) 和 h(x') 是否在根异侧（概率事件）
  - 驱逐路径是否命中（随机事件）
  → 无法根据这些信息调整 delta

adaptive 允许：每轮 delta 基于之前观察决定
  → 条件不满足就跳过 → 条件满足就攻击
  → 17 个 epoch 内必然成功
```

## 五、FUPSU 为什么免疫

### 5.1 Layer 1 — 加密隐藏 IBLT 结构

```
CT = SE.Enc(K_t, IBLT)
→ 敌手看到密文，看不到 cell 分布
→ 即使 h_i 公开，也无法通过 cell 位置反推元素
```

### 5.2 Layer 2 — IBLT 的随机图论保证

```
IBLT failure：2-core 出现
概率：O(τ^{-k+2})，其中 τ ≥ n

对于 k = 5, m > c_k × n：
  failure ≤ 2^{-λ}（λ ≈ 40+）

这个概率只依赖 m/n 比率，不依赖：
  ✗ 输入的选择策略
  ✗ 元素分布
  ✗ 是否自适应选择
```

### 5.3 Layer 3 — PSU 超集输出

```
PSI：输出 = X ∩ Y → 可通过输出推断 membership
PSU：输出 = X ∪ Y → 敌手添加的元素无条件出现在输出
→ 无法通过 union 输出推断对方集合内容
```

## 六、IBLT vs Path-ORAM 本质对比

| 维度 | Path-ORAM | IBLT |
|------|----------|------|
| 结构 | 二叉树 + 层级 | 扁平 k×ℓ 矩阵 |
| 位置映射 | h 公开 + 路径概念 | h 公开 + 无层级 |
| 容量 | 局部：每路径 c 个 | 全局：m/n 比率 |
| 溢出条件 | 局部碰撞 c+1 个 | 全局 2-core（随机图论事件） |
| 加密状态 | 明文 | 密文（IND-CPA） |
| 自适应安全 | ❌ | ✅ |

## 七、LMR+26 的修复

### ASE（Affine Set Encoding）

OKVS 的放松版本，关键性质：编码不暴露元素位置。

| 变体 | 自适应安全？ | 原因 |
|------|------------|------|
| Tree | ❌ | 继承 Path-ORAM 弱点 |
| OKVS | ✅ | 固定大小 ASE，仅支持 Add |
| Cuckoo | ✅ | Cuckoo hash + ASE，支持 Add & Del |
