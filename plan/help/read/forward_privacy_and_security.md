# 前向隐私定义、F-PSU 满足性、安全框架分析

## 一、前向隐私（Forward Privacy）的定义

F-PSU 论文中前向隐私有**两层定义**，分别对应 UC 框架和游戏框架：

### 1.1 理想功能层面：F_fpsu 中的前向隐私

在 UC 理想功能 `F_fpsu` 中，前向隐私通过**腐败时泄露的信息量**来刻画（§5.4）：

```
F_fpsu 与 P0, P1 交互，维护每方状态 (X_b, K_b, t_b)。

腐败规则（核心）：
- 当敌手在 epoch τ 自适应腐败 P_b 时：
  → 发送 (X_b, K_b) 给模拟器 S
  → 不发送 {X_b^{(<τ)}, K_b^{(<τ)}}  ← 前向隐私的关键
```

**前向隐私的本质**：敌手在 τ 时刻腐败某方后，能获得**当前**集合与密钥，但**无法获得历史epoch 的集合和密钥**。这意味着历史上的 PSU 输出仍然是保密的。

### 1.2 游戏层面：FP-IND 游戏（§5.4.2）

论文同时给出了一个具体的、可归约的游戏定义：

```
FP-IND Game（前向隐私不可区分游戏）：

1. Challenger 采样 b ← {0,1}, K_0 ← {0,1}^κ, X_0 ← ∅
2. For t = 1 to T:
   - 敌手选择 (Δ⁺_t, Δ⁻_t)
   - Challenger 更新 X_t，计算 K_t ← PRF(K_{t-1})
   - 加密 CT_t ← ChaCha.Enc(K_t, Encode(X_t))
   - 安全擦除 K_{t-1}
3. 敌手选择腐败 epoch τ，获得 (X_τ, K_τ)
4. If b = 0: 敌手收到真实密文 {CT_t}_{t<τ}
   If b = 1: 敌手收到随机串 {R_t}_{t<τ}
5. 敌手输出 b'，若 b' = b 则获胜

优势：Adv^{FP-IND}_A(κ) = |Pr[b' = b] - 1/2| ≤ negl(κ)
```

**直观理解**：腐败前的历史 IBLT 密文与随机串不可区分，即历史集合信息不泄露。

### 1.3 两种定义的定位与主流对比

| 维度 | 本文（F-PSU） | FUPSI (WZC+24, TIFS) | 主流做法 |
|------|-------------|---------------------|---------|
| 框架 | UC + Game 双重 | Game-based only | 多数 PSU 论文用 UC |
| 自适应腐败 | 支持 | 不支持 | 半诚实通常不支持 |
| 前向隐私定义方式 | 腐败时历史状态不可见 | 密钥更新后历史不可解密 | 与 Signal 双棘轮类似 |
| 泄露刻画 | 明确泄露 |Δ⁺|, |Δ⁻| | 仅泄露总集合大小 | 多数论文不讨论泄露 |

**结论**：F-PSU 的前向隐私定义是**完备且主流**的——UC 框架是密码学黄金标准，FP-IND 游戏提供了具体可操作的归约目标。双重定义覆盖了理论（UC）和实践（Game）两个层面。

## 二、F-PSU 如何满足前向隐私

### 2.1 核心机制：PRF 密钥链 + 即时擦除

```
初始化: K_0^C ←$ {0,1}^κ

每轮 epoch t 结束:
  K_t^C ← PRF(K_{t-1}^C, "fpsu-evolve-C")   // 密钥演化
  CT_stored ← ChaCha.Enc(K_t^C, N_C, IBLT_C)  // 加密当前 IBLT
  secure_erase(K_{t-1}^C)                      // 安全擦除旧密钥
```

**安全性论证**（两步）：

1. **PRF 单向性**：给定 K_t = PRF(K_{t-1})，敌手无法计算 K_{t-1}（否则就攻破了 PRF 的单向性）。
2. **密文不可区分性**：没有 K_{t-1}，敌手无法解密 CT_{t-1}（ChaCha20 是 IND-CPA 安全的流密码）。因此历史密文与随机串不可区分 → 满足 FP-IND。

### 2.2 关键设计决策：XOR 刷新已删除的原因

早期版本曾考虑用 `CT' ← CT ⊕ PRG(K_t)` 来"刷新"密文，但这与**前向隐私矛盾**：
- XOR 刷新意味着用 K_t 去处理历史密文（旧 IBLT 用新密钥重新加密）
- 这要求保存 K_{t-1} 直到完成刷新 → 延迟了密钥擦除
- 实际上创造了密钥共存窗口，削弱了前向保障

**最终方案**：每轮独立加密 + 即时擦除，不做任何历史数据刷新。存储的是用 K_t 加密的**当前** IBLT，而非 K_{t-1} 加密的历史 IBLT。

### 2.3 前向 vs 后向安全（Forward vs Post-Compromise）

F-PSU 的 PRF 链设计与 Signal 协议的对称棘轮有相同的取舍：

```
PRF 链: K_1 → K_2 → K_3 → ... → K_t → ...

腐败 K_τ 后：
  ✅ 保护: 历史密钥 {K_{<τ}}（PRF 不可逆向计算）
  ❌ 暴露: 未来密钥 {K_{>τ}}（PRF 可正向计算）
```

这是**前向安全**（Forward Security）而非**后向/自愈安全**（Post-Compromise/Healing Security）：
- 后向安全需要每轮注入新鲜随机性（如 DH 密钥交换或新鲜 Nonce） → 增加通信
- F-PSU 的纯本地 PRF 链实现了**零额外通信**的前向隐私，这是通信效率与安全性的 trade-off

## 三、安全框架分析

### 3.1 框架总览

```
安全模型: UC 框架（Universally Composable）
敌手模型: PPT 半诚实自适应（Semi-honest Adaptive）
混合模型: (F_OPRF, F_OT)-hybrid
密码假设:
  - ChaCha20 is a secure PRG
  - HMAC-SHA256 is a secure PRF
安全目标: Π_fpsu UC-realizes F_fpsu
总优势: T · (Adv^PRF + Adv^PRG) ≤ negl(κ)
```

### 3.2 与主流的符合程度

| 方面 | 本文 | 主流标准 | 符合度 |
|------|------|---------|-------|
| **UC 框架** | ✅ 完整 UC 证明 | PSU 论文主流（Piske & Trieu EC26, ePSU USENIX'25） | ★★★★★ |
| **半诚实** | ✅ 半诚实 | 多数 PSU/PSI 论文首版，理解半诚实后再做 malicious | ★★★★☆ |
| **自适应腐败** | ✅ 支持 | **罕见**，大多数半诚实论文只用静态腐败 | ★★★★★ |
| **混合模型** | ✅ (F_OPRF, F_OT) | 标准做法，与 Piske & Trieu 一致 | ★★★★★ |
| **具体假设** | ChaCha20 + HMAC | 无 ROM 假设，仅标准模型假设 | ★★★★☆ |
| **泄露可证明** | ✅ 下界定理 | 极少数论文讨论泄露下界 | ★★★★★ |

**主要局限**：
- 半诚实安全 → 恶意安全需要额外机制（如 IBLT 完整性保护、恶意 OT/OPRF）
- 仅前向安全 → 需要后向安全的场景不适用

### 3.3 安全证明结构（Hybrid Proof）

证明通过 **4 个混合模拟器** 的逐步过渡完成：

```
S_0 (Real World)
  │  运行真实协议，使用真实 P1 输入
  │
  ▼  PRF 安全性（T · Adv^PRF）
S_1 (Random Keys)
  │  P1 的 PRF 链密钥替换为独立随机密钥
  │  与 S_0 不可区分（否则攻破 PRF）
  │
  ▼  PRG 安全性（T · Adv^PRG）
S_1.5 (Dummy Ciphertexts)
  │  P1 加密的 IBLT 替换为全零串加密
  │  与 S_1 不可区分（否则攻破 PRG）
  │
  ▼  信息论等效（UnionPeel 输出取决于 IBLT_U）
S_2 (Simulated World)
  │  使用 Ỹ_t = X_{U,t} \ X_t 替代真实 Y_t
  │  仅需 F_fpsu 的泄露信息
  └─ 与 S_1.5 完全相同分布
```

### 3.4 UC 组合定理（模块化分解）

论文将协议分解为两个子模块：

1. **π_epoch**：单 epoch 内执行 Round 1-2（不含密钥演化），UC-实现静态 PSU 理想功能 F_psu（来自 Piske & Trieu）
2. **F_fwKey**：密钥管理子协议（PRF 链 + 加密 + 擦除），实现前向私密密钥管理

通过 **Per-Epoch UC Lemma** 和 **UC Composition Lemma**：
```
π_epoch UC-realizes F_psu  ∧  Key-Mgmt UC-realizes F_fwKey
    ⇒  Π_fpsu UC-realizes F_fpsu
```

这种模块化分解是论文的**隐藏优势**：任何改进的静态 PSU 协议（如恶意安全版）可以直接替换 π_epoch，保持前向隐私特性不变。

### 3.5 已知攻击的排除

| 攻击 | 来源 | 为何不适用 |
|------|------|-----------|
| OKVS 过拟合 | Han et al. | 仅适用于恶意接收者场景，F-PSU 是半诚实模型 |
| 对称棘轮 TMTO | Mattsson | PRF 链为单链无分支，TMTO 复杂度 O(2^κ = 2^256)，不可行 |
| 通信量侧信道 | 本文下界定理 | 已证明个体 Δ 计数泄露是任何 O(\|Δ\|) 协议都无法避免的 |

## 四、总结

1. **前向隐私定义**：双层定义（F_fpsu 理想功能腐败规则 + FP-IND 不可区分游戏），符合主流 UC 标准且更为精确
2. **满足机制**：PRF 链 K_{t+1}=PRF(K_t) + ChaCha20 加密 + 即时密钥擦除，零额外通信开销
3. **安全框架**：UC 框架，半诚实自适应敌手，(F_OPRF, F_OT)-hybrid 模型，通过 4 步 hybrid proof + 组合定理完成证明
4. **主流符合度**：整体符合且超越（自适应腐败一般不讨论）；半诚实 vs 恶意是主要限制但有明确升级路径
